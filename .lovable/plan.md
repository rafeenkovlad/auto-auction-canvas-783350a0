## Что найдено в интернете

Просмотрел доступные варианты:

1. **SVG Repo** (`svgrepo.com/svg/166983/car-top-view`) — открытая лицензия (CC0/MIT), готовые векторы видов автомобиля сверху/сбоку/спереди/сзади. Можно скачивать и адаптировать свободно.
2. **Japanese auction body maps** (integrityexports.com, beforward, jpsheet) — это эталонный визуальный язык для отчётов об осмотре: четыре проекции на одном листе (сверху + 3 боковые), помеченные зоны с буквенно-цифровыми кодами (A1, B2, U, X, W). Именно этот стиль ожидается в отчёте по аукционному автомобилю.
3. **Interactive Clickable Car Diagram** (freelancertajulrasel) — платные шаблоны (~$30), брать их нельзя, но структура зон (68 элементов: bumper, fender, door, quarter panel, hood, roof, trunk и т.д.) — хороший справочник по неймингу.
4. **GitHub `autopainting-calc`** — open-source SVG с разметкой панелей кузова сверху/сбоку, MIT-подобная, можно использовать как референс силуэта.

Чистого готового SVG с 5 проекциями под свободной лицензией нет, поэтому соберём свой набор на основе открытых силуэтов из SVG Repo и визуального языка японских аукционных карт.

## План

### 1. Подготовка SVG-силуэтов

Создать `src/components/body-schema/` с 4 отдельными SVG-компонентами (отказываемся от 5-го вида — на японских картах его нет, и это нагляднее):

- `TopView.tsx` — седан сверху: капот, крылья (FL/FR), двери (FL/FR/RL/RR), задние крылья (RL/RR), крышка багажника, крыша, бамперы. Контуры — на основе SVG Repo `car-top-view`, перерисованные path'ами с зонами `<path data-zone="...">`.
- `FrontView.tsx` — спереди: бампер, решётка, капот, фары L/R, лобовое стекло, крыша.
- `RearView.tsx` — сзади: бампер, крышка багажника, задние фары L/R, заднее стекло.
- `SideView.tsx` — единый боковой вид с зеркальным переключателем (props `side: 'left' | 'right'` через `scaleX(-1)`), вместо двух отдельных файлов. Зоны: переднее крыло, передняя дверь, задняя дверь, заднее крыло, порог, переднее/заднее колесо, окна, крыша.

Все пути с `fill="var(--zone-bg)" stroke="currentColor"` и атрибутами `data-zone` / `data-label`, чтобы родительский компонент мог подсвечивать их по статусу.

### 2. Переделка `CarBodySchema.tsx`

- Заменить inline-SVG примитивы на импорты новых компонентов выше.
- Переключатель видов: `Сверху | Спереди | Сзади | Слева | Справа` (5 кнопок, но `Слева`/`Справа` рендерят один `SideView` с разными `side`).
- Сохранить текущую логику: цвет зоны по статусу (`ok` / `minor` / `serious` / `none`), hover/click → подсветка в списке замечаний и открытие `ElementViewer`.
- Добавить мелкие подписи зон (A/B/C коды) поверх SVG в стиле японских листов — опционально, через `<text>` внутри SVG.

### 3. Маппинг зон API → панелей

Расширить словарь зон в `CarBodySchema.tsx`, чтобы покрыть детализированные имена (front_left_fender, front_left_door, rear_left_door, rear_left_quarter, hood, roof, trunk, front_bumper, rear_bumper, front_left_headlight, rear_left_taillight, windshield, rear_window, rocker_left, …). Сопоставление с `_sectionKey` элементов API — через нормализацию строк (как сейчас).

### 4. Стили

В `src/styles.css` оставить уже добавленные `--zone-*-bg`. Добавить тонкие `stroke-width` и `stroke-linejoin: round` глобально для `.body-schema svg path`, чтобы силуэты выглядели аккуратно на любом размере.

### 5. Что НЕ трогаем

- `src/routes/index.tsx` — кроме случая, если меняется API компонента (не меняется).
- `MediaGallery.tsx`, `ElementViewer.tsx`, `report.functions.ts` — без изменений.

## Технические детали

- Силуэты перерисовываются вручную как набор `<path d="...">` на основе референсных SVG из SVG Repo (CC0). Не копируем чужой код 1:1, а воспроизводим геометрию — это легально и даёт чистые контуры под наши data-атрибуты.
- ViewBox единый, например `0 0 400 800` для top/side и `0 0 400 280` для front/rear, чтобы переключение видов не «прыгало».
- Никаких внешних зависимостей — всё инлайн SVG в TSX, без `<img src>`.
