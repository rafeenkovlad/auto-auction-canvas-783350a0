## Цель

Сейчас вкладки схемы осмотра выглядят по-разному:

- **Кузов** — полноценный layout: схема слева, сайдбар справа с легендой и подробным списком замечаний (заметка, теги, ЛКП), плавающий tooltip с названием зоны при наведении, подсказка снизу.
- **Силовые, Стёкла, Колёса, Освещение** — простой одноколоночный layout: схема сверху, под ней короткий список «Замечания» с одной строкой на элемент (без тегов/заметки), без легенды.
- **Салон** — вообще сетка кнопок без схемы.

Нужно, чтобы все вкладки наследовали интерфейс «Кузова»: одинаковая компоновка, одинаковый сайдбар с замечаниями, легенда, hover-label, подсказка снизу.

## План

### 1. Вынести общий shell — `src/components/SchemaShell.tsx`

Новый компонент принимает:

- `canvas: ReactNode` — содержимое схемы (SVG/картинка/группа панелей)
- `hoverLabel?: string` — название наведённой зоны, рендерится плавающим бейджем над низом канваса
- `damaged: InspectionElement[]` — список элементов с замечаниями
- `zoneLabelFor: (el) => string` — как назвать элемент в списке (по типу зоны/elementType)
- `onElementClick?`, `onItemHover?: (id|null) => void` — чтобы наведение в списке подсвечивало зону
- `header?: ReactNode` — слот для доп. контролов (например, переключатель «Левая/Правая сторона» во фрейме)
- `footerHint?: string` — текст подсказки снизу (по умолчанию как в кузове)
- `canvasBackground?: boolean` — обернуть канвас в «панель» с подложкой (как в кузове). Для фотоподложек освещения — отключить.

Внутри:

```
┌─ header (опционально) ────────────────────┐
├─ grid md:grid-cols-[1fr_minmax(180px,240px)]
│  ┌─ canvas wrapper ─────┐  ┌─ sidebar ──┐
│  │  {canvas}            │  │ Легенда     │
│  │  hover label badge   │  │ ─────────   │
│  └──────────────────────┘  │ Замечания   │
│                            │ (полные     │
│                            │ карточки)   │
│                            └─────────────┘
└─ footer hint ─────────────────────────────┘
```

Карточка замечания — тот же markup, что в `CarBodySchema` (название, ЛКП-бейдж, заметка, теги).

### 2. Рефакторинг `CarBodySchema`

Извлечь канвас-часть (TopView + hover label) и список зон, передать в `SchemaShell`. Удалить локальный layout — он переезжает в shell. Это эталон, на котором тестируем shell.

### 3. Привести остальные схемы

- **FrameSchema** — переключатель сторон уйдёт в `header`, SVG — в `canvas`. Список «Замечания» удалить (его рендерит shell).
- **WheelsSchema** / **GlassSchema** — заменить `ZoneSchema` на shell + локальный inline-канвас (или оставить ZoneSchema только для отрисовки зон без сайдбара).
- **LightingSchema** — двухпанельный layout (перед+зад) уезжает целиком в `canvas`, всё остальное (легенда, замечания, подсказка) даёт shell. Убрать локальный блок damaged.
- **Interior (PlaceholderBoard в SchemaTabs)** — вынести в отдельный `InteriorSchema.tsx`. Канвас — сетка кнопок (как сейчас) или иллюстрация салона-плейсхолдер; сайдбар с замечаниями добавит shell. Если данных нет — shell покажет пустое состояние.

### 4. Общие хелперы статусов

`statusOf`, `fillFor`, `strokeFor` уже задублированы в каждой схеме. Перенести в `src/lib/report.utils.ts` (там уже есть `getElementStatus/statusFill/statusStroke` — используем их везде).

### 5. Сопоставление элемент → название зоны

Каждая схема экспортирует свою таблицу алиасов (`ELEMENT_ZONE_LABEL`) и передаёт `zoneLabelFor` в shell, чтобы карточки в сайдбаре читались человекочитаемо («Передняя левая фара» вместо `front_left_headlight`).

## Что НЕ трогаем

- Внутренности SVG конкретных схем (формы зон, координаты, фотоподложки) — только обёртку.
- Логику табов в `SchemaTabs` (вкладки, индикаторы статусов).
- Модалку детального просмотра элемента, которая открывается по `onElementClick`.

## Технические детали

- Файлы: новый `src/components/SchemaShell.tsx`; правки в `CarBodySchema.tsx`, `FrameSchema.tsx`, `WheelsSchema.tsx`, `GlassSchema.tsx`, `LightingSchema.tsx`, `SchemaTabs.tsx`; новый `InteriorSchema.tsx`.
- `ZoneSchema.tsx` — оставляем как «дешёвый» движок отрисовки зон без обёртки; либо упрощаем до экспорта `statusOf/fillFor/strokeFor` (этих хелперов больше нет — переезжают в utils) и удаляем целиком после миграции `Wheels`/`Glass` на свой inline-канвас.
- Никаких изменений в API/данных — всё на уровне презентации.
