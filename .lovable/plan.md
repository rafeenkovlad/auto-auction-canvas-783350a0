## Схема кузова

Используем готовый open-source SVG `car-views` (top/front/rear/left/right проекции) из публичных damage-inspection шаблонов. Подгружаем как inline-SVG-компонент (`src/components/CarBodySchema.tsx`), каждой `<path>` детали назначаем `data-zone="hood|roof|front_bumper|...`. Маппинг `elementType → zone` уже есть в коде.

- Состояния zone: `ok` (зелёный фон), `minor` (жёлтый), `major` (красный), `none` (нейтральный серый).
- Hover/click по зоне → подсветка соответствующей карточки в списке замечаний справа от схемы и открытие `ElementViewer`.
- Переключатель видов — 5 кнопок-табов, рендерим только активную проекцию.

## Технические детали

1. **Новый компонент** `src/components/CarBodySchema.tsx`:
   - props: `elements: EnrichedElement[]`, `onSelect: (idx) => void`
   - внутри 5 inline SVG, переключаемых через `useState<'top'|'front'|'rear'|'left'|'right'>`
   - применяет цвета через `data-status` атрибут + CSS

2. **Новый компонент** `src/components/MediaGallery.tsx`:
   - принимает все `elements` с файлами, группирует по `_sectionKey`/типу медиа
   - табы с подсчётом, сетка миниатюр (image + video preview через существующий `VideoThumb`)

3. **`src/routes/index.tsx`**: переписываем `AuctionSheetPage` под новую структуру. Header — узкая полоса. Hero — фото авто (первое изображение из `carStep.files` или `listingFile`) + краткие характеристики.

4. **Дизайн-токены**: добавляем мягкие фоновые оттенки для статусов (`--zone-ok-bg`, `--zone-warn-bg`, `--zone-bad-bg`) в `src/styles.css`.

5. **Что НЕ показываем** (нет в API): ФИО специалиста, % уверенности, общая оценка X/5, оценки по категориям, итоговое решение, timeline с временными метками.

6. **Фон страницы**: меняем на светло-серый/градиент как в мокапе, панели делаем белыми с мягкой тенью.
