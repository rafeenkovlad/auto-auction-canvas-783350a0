# План: выравнивание зон схемы кузова по референсу

Файл: `src/components/CarBodySchema.tsx`, функция `TopView`.
Фоновая подложка `car-top.png` (1024×1536) и набор зон/ID остаются прежними — меняется только геометрия `<path d=...>` для каждой зоны, чтобы повторять пунктирные контуры с присланного референса.

## Соответствие подписей референса и внутренних ID

| Подпись на референсе | ID в коде |
|---|---|
| hood | `hood` |
| front_bumper | `fbumper` |
| left_front_wing / right_front_wing | `flfender` / `frfender` |
| front (лобовое) | `windshield` |
| front_left / front_right (передние боковые окна) | `lfwin` / `rfwin` |
| left_front_door / right_front_door | `fldoor` / `frdoor` |
| left_rear_door / right_rear_door | `rldoor` / `rrdoor` |
| rear_left / rear_right (задние боковые окна) | `lrwin` / `rrwin` |
| left_rear_wing / right_rear_wing | `rlfender` / `rrfender` |
| roof (центр + 2 нижних подписи) | `roof` (одна крыша; нижние подписи на референсе — те же rear quarters) |
| side_left / side_right (пороги) | `lthresh` / `rthresh` (зоны добавить — сейчас отрисовываются только в боковых видах) |
| trunk | `trunk` |
| rear_bumper | `rbumper` |
| (заднее стекло) | `rear_window` |

## Геометрические правила (под силуэт car-top.png, bbox ≈ x:260–770, y:95–1265)

```text
                    y≈95  ┌─ front_bumper: тонкая дуга поперёк носа
                  y≈145  ─┤
                          │  hood: трапеция, сужается к носу,
                          │        нижняя кромка совпадает с верхом windshield
                  y≈470  ─┤
                          │  windshield: трапеция, расширяется назад
                  y≈600  ─┤
                          │  roof: прямоугольник по центру кабины
                  y≈1020 ─┤
                          │  rear_window: трапеция, сужается назад
                  y≈1140 ─┤
                          │  trunk: прямоугольник до заднего бампера
                  y≈1240 ─┤
                          │  rear_bumper: тонкая дуга поперёк кормы
                  y≈1280 ─┘

  По сторонам каждый ярус:
    front_wing  — L-образный клин от носа до A-стойки
    front_door  — прямоугольник от A- до B-стойки
    rear_door   — прямоугольник от B- до C-стойки
    rear_wing   — клин от C-стойки до кормы
    threshold (side_*) — узкая полоса под дверями вдоль порога
    front_win / rear_win — узкие боковые трапеции окон между дверной кромкой и крышей
```

Все соседние зоны делят ровно одну общую кривую/прямую, чтобы:
- не было визуальных пересечений;
- не оставалось зазоров;
- все контуры лежали внутри bbox силуэта (никаких выходов в белое поле).

## Что меняется в коде

1. `TopView`: заменить блок `<path ... overlay(...)>` (строки ~368–390) на новый набор путей, построенный по правилам выше.
2. Добавить пути для `lthresh` и `rthresh` (узкие полосы порогов вдоль дверей) — сейчас их нет в TopView.
3. Обновить словарь `CENTERS` так, чтобы маркеры замечаний попадали в новые центры зон (включая `lthresh`/`rthresh`).
4. Сохранить порядок отрисовки: бампера/крылья → капот/багажник → стёкла/двери → крыша поверх дверей, окна поверх крыши (как сейчас), маркеры замечаний — последними.

## Что НЕ меняется

- Фоновая картинка `car-top.png` и её координатная сетка 1024×1536.
- Логика `zoneProps`, `overlay`, маркеры, цвета, hover, клики.
- Боковые/передний/задний виды (`FrontView`, `RearView`, `SideView`).
- Маппинг `ELEMENT_ZONE` и подписи `ZONE_LABEL`.

## Проверка после правки

- Открыть превью на `/` с тестовым отчётом, переключить вид «Сверху».
- Визуально пройти по каждой зоне (hover) — контур должен ложиться поверх соответствующей детали авто и не залезать на соседние.
- Проверить, что маркеры повреждений рисуются по центру своих зон.
